# 0. 사용자 시나리오

## S1. 방문자 - 블로그 목록 탐색
- 경로/화면: `/blog` 리스트 페이지.
- 목표: 최신 포스트를 검색·필터링하며 카드 형태로 탐색.
- 주요 플로우: 페이지 진입 → 백엔드 `/posts` 목록 호출(검색어/태그/페이지네이션 쿼리) → 카드/페이지네이션/태그 필터 UI 업데이트.
- 성공 조건: 게시글 목록과 총 개수 노출, 검색·태그 선택 시 목록 갱신, 결과 없음 안내.

## S2. 방문자 - 포스트 상세 열람 및 공유
- 경로/화면: `/blog/:id` 상세 페이지.
- 목표: 포스트 본문(마크다운 렌더링), 목차(Table of Contents), 관련 글, 공유 기능 확인.
- 주요 플로우: `/posts/:id` 조회 → 본문 렌더링 및 목차 생성 → 공유 버튼/관련 글 영역 노출.
- 성공 조건: 제목/요약/본문/메타 정보(카테고리, 태그, 작성일, 읽기 시간 등) 표시, 공유·관련 글 링크 정상 동작.

## S3. 방문자 - 댓글 작성 및 스레드형 대화
- 경로/화면: 포스트 상세 하단 댓글 섹션.
- 목표: 댓글 작성, 대댓글/수정/삭제를 통한 스레드형 대화.
- 주요 플로우: 폼에 내용 입력 → `/posts/:postId/comments`로 등록 → 목록 새로고침 및 중첩 구조 표시 → (선택) 수정·삭제·답글.
- 성공 조건: 작성/수정/삭제 결과가 즉시 UI에 반영되고, 중첩 댓글이 계층적으로 노출.

## S4. 관리자 - 포스트 관리(CRUD)
- 경로/화면: `/admin/posts`, `/admin/posts/new`, `/admin/posts/:id/edit`.
- 목표: 관리자 권한으로 포스트 생성, 수정, 삭제 및 상태 관리.
- 주요 플로우: 관리자 대시보드 접근 → 목록 조회 → 새 글 작성(마크다운 에디터/태그/카테고리/상태 설정) → 저장 또는 게시 → 필요 시 수정·삭제.
- 성공 조건: CRUD 요청이 성공하고 목록/상세가 갱신되며, 인증 토큰(개발용 `dev-token` 포함)이 유효하게 처리.
