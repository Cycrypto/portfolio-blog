# 3. 현재 구현 및 미완성 사항

## 구현 현황
- **백엔드**: Post/Tag/Comment CRUD와 태그 랭킹, 게시 상태 필터, Swagger 데코레이터, 개발용 `POST /dev-token`(admin JWT) 제공.
- **프론트엔드**: 
  - `/blog` 목록/검색/태그 필터/페이지네이션, `/blog/:id` 상세(마크다운 렌더링, 목차, 공유, 관련 글), 댓글 UI(작성·수정·삭제·대댓글).
  - 관리자 `/admin/posts` 목록/삭제, `/admin/posts/new` 작성(마크다운 에디터, 태그/카테고리/상태 설정), `/admin/posts/:id/edit` 수정 흐름 구현.
- **통신**: `apiRequest` 유틸로 JWT 헤더 포함 옵션 지원(기본 인증 필요), 목록 페이지는 공개 fetch를 사용해 비인증 조회도 동작.

## 미완성/주의 사항
- **댓글 모더레이션 UI 보완**: `/admin/comments`로 기본 조회/삭제는 가능하나, 검색/필터/다중 선택 등 고급 기능은 추가 여지.
- **문서화/README**: 루트 및 백엔드 README는 기본 템플릿 수준으로 실제 서비스 정보, 실행 방법(DB 설정 등) 보완 필요.

### 프론트 UI는 있지만 구현이 비어있거나 개선 여지
- **관리자 포스트 검색/필터**: 검색·상태 필터가 실제 목록 필터링으로 연결됨(구현 완료).
- **프로젝트 관리 UI**: `/admin/projects` 목록/신규/수정/삭제가 프로젝트 API와 연동됨.
- **프로필 편집 화면**: `/admin/profile`이 백엔드 API와 연동되어 저장·이미지 업로드가 동작.
- **댓글 답글 작성자 정보**: 모든 댓글/답글 작성이 입력한 이름/이메일을 사용하도록 수정됨.
- **파일 업로드 버튼**: 프로필/포스트/프로젝트 모두 업로드 엔드포인트를 통해 이미지 업로드 가능. (다중/용량 제한 등 추가 고려 필요.)

### 필요한 화면이 없거나 연동이 누락된 부분
- **인증/로그인 UI**: `/admin/login` 및 레이아웃 리다이렉트 구현. 추후 일반 사용자 로그인/갱신 흐름 확장 필요.
- **프로젝트/프로필 퍼블릭 완성도**: 퍼블릭 프로젝트 상세/목록은 API 연동 완료. 필터/검색/SEO 메타 등 추가 개선 가능.
- **댓글 관리자/모더레이션 UI**: `/admin/comments` 기본 삭제/조회는 가능하나, 태그/검색/페이지네이션 등 확장 필요.

## 테스트 상태
- 자동화 테스트/CI 설정 언급 없음. 현재 변경 사항에 대해 테스트는 실행하지 않았으며, DB 연결 정보 설정 후 수동 검증 필요.
