# 5. 프론트엔드 목업/하드코딩 인벤토리

프로덕션 연동 없이 남아 있는 목업·하드코딩 데이터와 placeholder를 정리했습니다. 실제 API/콘텐츠로 대체하거나 디자인 결정이 필요합니다.

## 홈/퍼블릭 영역
- `frontend/app/page.tsx`
  - 프로젝트/경험 카드 데이터가 상수로 하드코딩, 이미지도 `/placeholder.svg?...` 사용.
  - 소개 섹션 이미지/텍스트, “이력서 다운로드” 버튼 등이 정적 값.
- `frontend/components/blog/author-card.tsx`
  - 아바타 기본값이 `/placeholder.svg`; 실제 프로필 이미지로 교체 필요.

## 관리자 대시보드
- `frontend/app/admin/page.tsx`
  - 통계(`totalPosts`, `totalProjects`, `totalViews`, `totalComments`)와 `recentPosts`, `recentProjects`가 모두 목업 상수.

## 프로젝트 관련 컴포넌트
- `frontend/components/project/project-gallery.tsx`
  - 이미지 미제공 시 `/placeholder.svg` 대체.
- `frontend/components/project/project-card.tsx`
  - `image` 누락 시 `/placeholder.svg` 사용.

## 블로그/댓글
- `frontend/app/blog/[id]/page.tsx`
  - AuthorCard avatar 기본값 `/placeholder.svg?height=100&width=100`.
- `frontend/components/blog/CommentSection.tsx`
  - 댓글 아바타 기본값 `/placeholder.svg`.

## 프로필 기본값
- `frontend/lib/api/services/profile.ts`
  - `getDefaultProfile()`에 이름/타이틀/기술 스택/`profileImage: /placeholder.svg` 등 하드코딩 초기값 존재.

## 기타 참고
- placeholder 이미지는 UI 기본값으로 오류는 아니나, 운영 시 실이미지로 교체 필요.
- 입력 필드 placeholder(예: contact-form, ProfileEditor 등)는 UX용 안내 문구이며 치환 여부는 정책에 따라 결정.
