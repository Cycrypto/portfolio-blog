# 2. 기능 명세서

## 블로그 방문 기능
- 목록 조회: 키워드 검색, 태그 필터, 페이지네이션을 통한 포스트 카드 렌더링.
- 태그 랭킹: `/tags/top`으로 상위 태그 표시(필터/추천 목적).
- 상세 보기: 제목·요약·본문(마크다운 렌더링), 카테고리/태그/작성일/읽기 시간/조회수/댓글수 메타 정보 노출.
- 부가 UX: 목차(Table of Contents) 자동 생성, 공유 버튼, 관련 글 영역(`getPosts` 기반 근접/랜덤 추천).

## 댓글 기능
- 작성: 익명 입력 기반의 본문/작성자 이름/이메일로 댓글 생성, 선택적 부모 ID로 스레드 지원.
- 조회: 대댓글을 포함한 중첩 구조 반환, 삭제된 댓글 `isDeleted` 표기.
- 수정·삭제: UI 상 편집/삭제 버튼 제공. 컨트롤러는 인증 요구로 문서화되어 있으나 가드 미적용 상태(현재 누구나 호출 가능).

## 관리자 기능
- 목록 관리: `/admin/posts`에서 전체 포스트를 테이블로 조회하고 새로고침/검색 UI 제공.
- 작성: `/admin/posts/new`에서 마크다운 에디터, 요약, 카테고리/태그, 상태(임시저장/게시/예약), 대표 이미지 설정 후 저장.
- 수정: `/admin/posts/:id/edit`에서 변경 필드만 전송하여 업데이트, 성공/에러 피드백 제공.
- 삭제: 목록/상세에서 삭제 버튼 호출. 백엔드 `JwtRoleGuard`로 보호되며, 개발·테스트용 `POST /dev-token` 엔드포인트로 JWT 발급 가능.

## 기술 스택 및 통신
- 프론트엔드: Next.js(앱 라우터), Tailwind UI 구성요소, `NEXT_PUBLIC_API_URL`(기본 `http://localhost:3002`) 기반 REST 호출.
- 백엔드: NestJS + TypeORM(PostgreSQL), Post/Tag/Comment 엔티티 및 Swagger 데코레이터 포함. `synchronize: true`로 로컬 개발 DB 스키마 자동 동기화.
